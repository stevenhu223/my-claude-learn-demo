# 财务管家 - 部署与运维手册

---

## 一、项目概述

财务管家是一套完整的个人财务管理解决方案，包含四个主要组件：

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| Web 前端 | HTML + Tailwind CSS + JavaScript | 面向用户的网页端应用 |
| Node.js 后端 | Node.js + Express + MySQL | Web 前端的数据接口服务 |
| Java 后端 | Java 21 + Spring Boot + MyBatis Plus | 移动端和高级功能的数据接口 |
| UniApp 应用 | Vue 3 + uni-app | 跨平台移动端应用（微信小程序/H5） |

## 三、服务器要求

### 最低配置

- CPU：2核
- 内存：4GB
- 硬盘：20GB
- 操作系统：Windows Server 2019 / Ubuntu 20.04 / CentOS 7

### 推荐配置

- CPU：4核
- 内存：8GB
- 硬盘：50GB SSD
- 操作系统：Windows Server 2019 / Ubuntu 22.04

### 软件环境

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | ≥18.0 | 运行 Node.js 后端服务 |
| Java | 21 | 运行 Java 后端服务 |
| MySQL | 8.0 | 数据库服务 |
| Maven | 3.8+ | Java 项目构建工具 |
| PM2 | 最新版 | Node.js 进程管理器（可选） |

## 四、数据库配置

### 4.1 创建数据库

```sql
CREATE DATABASE finance_buddy CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 4.2 用户权限

```sql
-- 创建专用数据库用户（推荐）
CREATE USER 'finance_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON finance_buddy.* TO 'finance_user'@'localhost';
FLUSH PRIVILEGES;
```

### 4.3 初始化表结构

Java 后端会自动创建数据表（通过 MyBatis Plus），首次启动时无需手动执行 SQL。

## 五、部署步骤

### 5.1 Java 后端部署

```bash
# 1. 进入项目目录
cd 005.后端代码(java工程师)/finance-buddy-api

# 2. 修改数据库配置
# 编辑 src/main/resources/application.yml 中的数据库连接信息
# url: jdbc:mysql://localhost:3306/finance_buddy
# username: your_db_user
# password: your_db_password

# 3. 打包项目
mvn clean package -DskipTests

# 4. 启动服务
java -jar target/finance-buddy-api-1.0.0.jar

# 服务默认运行在 http://localhost:8080
# API 基础路径: http://localhost:8080/api
```

### 5.2 Node.js 后端部署

```bash
# 1. 进入项目目录
cd 003.前端代码(前端工程师)/backend

# 2. 安装依赖
npm install

# 3. 修改数据库配置
# 编辑 db/database.js 中的 MySQL 连接配置

# 4. 启动服务
node server.js

# 服务默认运行在 http://localhost:3000
# API 基础路径: http://localhost:3000/api
```

### 5.3 Web 前端部署

Web 前端是纯静态文件，可以部署到任意 Web 服务器：

```bash
# 文件位置
003.前端代码(前端工程师)/frontend/warm_finance/

# 部署方式
# - Nginx: 将文件放入 nginx/html 目录
# - Apache: 将文件放入 htdocs 目录
# - IIS: 创建新站点指向该目录

# 重要配置
# 如果后端服务不在本机，需要修改 js/api.js 中的 API 地址
```

### 5.4 UniApp 移动端部署

```bash
# 1. 使用 HBuilderX 打开项目
# 2. 修改 utils/api.js 中的 API 地址为实际服务器地址
# 3. 修改 manifest.json 中的 AppID 等配置
# 4. 运行到微信开发者工具（小程序）
# 5. 或发行到 H5（手机浏览器）
```

## 八、常见问题处理

### 问题1：服务无法启动

**可能原因：**

- 端口被占用
- 数据库连接失败
- 配置文件错误

**解决方法：**

```bash
# 检查端口占用
netstat -ano | findstr :8080  # Java
netstat -ano | findstr :3000 # Node.js

# 更换端口或关闭占用进程
```

### 问题2：数据库连接失败

**可能原因：**

- MySQL 服务未启动
- 用户名密码错误
- 数据库不存在

**解决方法：**

```bash
# 启动 MySQL
net start mysql

# 或
sudo systemctl start mysql
```

### 问题3：API 返回 401 未授权

**可能原因：**

- Token 过期
- 请求头格式错误

**解决方法：**

- 重新登录获取新 Token
- 检查 Authorization 请求头格式：`Bearer <token>`

### 问题4：移动端无法访问

**可能原因：**

- 服务器防火墙未开放端口
- CORS 配置问题

**解决方法：**

```bash
# 开放防火墙端口（Windows）
netsh advfirewall firewall add rule name="Finance API" dir=in action=allow protocol=tcp localport=8080

# 开放防火墙端口（Linux）
sudo ufw allow 8080/tcp
```
